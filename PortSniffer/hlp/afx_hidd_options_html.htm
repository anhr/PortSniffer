HTML page of the Port Sniffer Options dialog box

Apply options below for HTML protocol only.

The following options are allowed:

Replace the Host header

Replace the "Host" header field from the client's request to the server's URL. See the Listen Toolbar for details.

Has effect for some web servers only.

  • Example.

Here you want to connect to bonalink.com web server by the Port Sniffer from the localhost machine. Usually a web browser sends a request to web server, are similar to that:

GET / HTTP/1.1
Accept: image/gif, image/jpeg, image/pjpeg, image/pjpeg, application/x-shockwave-flash, application/x-ms-application, application/x-ms-xbap, application/vnd.ms-xpsdocument, application/xaml+xml, application/msword, */*
Accept-Language: en-US,fr;q=0.5
User-Agent: Mozilla/4.0 (compatible; MSIE 8.0; Windows NT 5.1; Trident/4.0; .NET CLR 1.1.4322; .NET CLR 2.0.50727; .NET CLR 3.0.4506.2152; .NET CLR 3.5.30729; .NET4.0C)
Accept-Encoding: gzip, deflate
Host: localhost
Connection: Keep-Alive

Line

Host: localhost

is incorrect. The correct line is similar to that:

Host: bonalink.com 

It happening because you navigate to localhost instead real webserver. My utility can replace this line if you mark the Replace the Host header check box.

Save the server's response to a file

Some web server's responses, such as images is binary and not readable. You can save server's responses to the file in appropiate format and open by associated application. Mark the Save the server's response to a file check box for it.

Folder for saved files

Specify the path to the folder, where you want to save the web server's response files.

Browse...

Opens the folder dialog, where you can choose a folder to save server's responses.
